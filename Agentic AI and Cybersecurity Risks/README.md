# Course Overview

Agentic AI systems introduce a fundamentally new cybersecurity paradigm.

Unlike traditional software or static AI models, agentic systems:
- act autonomously
- make decisions
- interact with tools, APIs, and other agents
- adapt dynamically over time

This creates a **new class of risks** that go beyond conventional cybersecurity.

Modern research and industry reports highlight that:

- Agentic AI systems behave like **“digital insiders”**, requiring insider-threat-style controls  [oai_citation:0‡ibm.com](https://www.ibm.com/think/insights/agentic-ai-security?utm_source=chatgpt.com)  
- They introduce **novel vulnerabilities across behaviour, tools, and runtime interactions**  [oai_citation:1‡McKinsey & Company](https://www.mckinsey.com/capabilities/risk-and-resilience/our-insights/deploying-agentic-ai-with-safety-and-security-a-playbook-for-technology-leaders?utm_source=chatgpt.com)  
- Security must evolve from static controls to **continuous monitoring, governance, and runtime enforcement**  [oai_citation:2‡Forbes](https://www.forbes.com/sites/anjanasusarla/2026/03/18/why-agentic-ai-requires-a-cybersecurity-governance-playbook/?utm_source=chatgpt.com)

# Learning Objectives

By completing this repository, you will:

- Understand the **attack surface of agentic AI systems**
- Identify **core vulnerabilities** (prompt injection, tool misuse, memory poisoning)
- Design **threat models for autonomous systems**
- Execute **red teaming workflows for agent-based architectures**
- Implement **defensive controls for secure deployment**
